Guia de Referência Flutter

Construa apps nativos para mobile, web e desktop a partir de uma única base de código.

O Princípio Fundamental: Tudo é um Widget

No Flutter, quase tudo é um Widget. Um widget pode ser um botão, um texto, um layout de colunas, um ícone, ou até mesmo a aplicação inteira. Eles são os blocos de construção da sua interface de usuário (UI).

A UI do seu app é construída como uma árvore de widgets. Cada widget é aninhado dentro de outro para criar layouts complexos. O Flutter então desenha essa árvore na tela.

Estrutura de um App Básico

Todo app Flutter começa com a função main(), que é o ponto de entrada. Dentro dela, a função runApp() é chamada para iniciar a árvore de widgets.

MaterialApp: Geralmente é o widget raiz do app. Ele configura o tema, as rotas de navegação e outros aspectos visuais seguindo as diretrizes do Material Design.

ThemeData: Define o tema visual do seu aplicativo, como cores primárias, fontes e estilos de componentes.

// main.dart import 'package:flutter/material.dart'; void main() { runApp(const MyApp()); } class MyApp extends StatelessWidget { const MyApp({Key? key}) : super(key: key); @override Widget build(BuildContext context) { return MaterialApp( title: 'Recipe Calculator', theme: ThemeData( primarySwatch: Colors.blue, ), home: const MyHomePage(title: 'Recipe Calculator'), ); } }

Anatomia de uma Tela: O Widget Scaffold

O widget Scaffold fornece uma estrutura visual padrão para as telas de um app Material Design. Ele facilita a adição de elementos comuns de UI.

  • appBar: A barra de ferramentas no topo da tela.
  • body: O conteúdo principal da tela, localizado abaixo da appBar.
  • floatingActionButton: Um botão flutuante, geralmente usado para ações primárias.
  • bottomNavigationBar: Uma barra de navegação na parte inferior da tela.
  • drawer: Um menu lateral que desliza a partir da borda.

Dica: Envolva seu Scaffold com um widget SafeArea para garantir que seu conteúdo não seja cortado por entalhes (notches) ou barras do sistema.

Montando Layouts: A Árvore de Widgets

Layouts são definidos aninhando widgets. A hierarquia visual do seu app é um reflexo direto de como sua árvore de widgets é estruturada no código.

Widgets de Layout Comuns:

  • Column: Organiza widgets filhos em uma coluna vertical.
  • Row: Organiza widgets filhos em uma linha horizontal.
  • Card: Cria um painel com cantos arredondados e sombra, seguindo o padrão Material.
  • ListView: Cria uma lista rolável de widgets. Ideal para exibir uma grande quantidade de itens.
  • Container: Um widget versátil para personalização, permitindo definir margens, preenchimento, bordas, cores de fundo, etc.

Stateless vs. StatefulWidget

Os widgets se dividem em duas categorias principais:

StatelessWidget

Widgets que não mudam depois de serem construídos. Seus valores são imutáveis. São usados para partes da UI que não precisam ser atualizadas dinamicamente, como um ícone, um texto fixo ou um rótulo.

StatefulWidget

Widgets que podem mudar durante o tempo de vida do app em resposta a interações do usuário ou recebimento de dados. Eles mantêm um "estado" que pode ser alterado. Quando o estado muda (usando setState()), o widget é redesenhado na tela.

// Exemplo: um contador que usa um StatefulWidget class Counter extends StatefulWidget { ... } class _CounterState extends State<Counter> { int _count = 0; void _increment() { setState(() { // Notifica o Flutter que o estado mudou e a UI precisa ser redesenhada _count++; }); } ... }
