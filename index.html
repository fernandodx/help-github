Guia Rápido de GitHub

Um passo a passo para colaborar em projetos usando Fork, Clone, Push e Pull Request.

O Fluxo de Trabalho Essencial

Este guia vai te ensinar o fluxo de trabalho mais comum para contribuir com um projeto no GitHub. A ideia é criar uma cópia sua do projeto (Fork), baixar essa cópia para o seu computador (Clone), fazer suas alterações, salvar (Commit), enviar de volta para o seu GitHub (Push) e, finalmente, sugerir que suas alterações sejam incluídas no projeto original (Pull Request).

Passo 0: Preparando o Ambiente

Antes de começar, certifique-se de que você tem o Git instalado no seu computador. Também é uma ótima prática manter seus projetos organizados. Nós recomendamos criar uma pasta chamada github na sua pasta de usuário.

Abra seu terminal (Git Bash no Windows, ou Terminal no Mac/Linux) e crie a pasta:

mkdir github
cd github

Todos os comandos a seguir devem ser executados dentro desta pasta github.

Passo 1: Fazendo o "Fork"

O "fork" é a sua cópia pessoal de um repositório. Você não pode editar o projeto original diretamente, então você cria uma cópia na sua conta do GitHub. Para isso, vá para a página do repositório que você quer copiar e clique no botão "Fork" no canto superior direito.

O GitHub pode perguntar para qual conta você quer fazer o fork. Escolha sua conta pessoal.

Passo 2: Clonando o Repositório (clone)

Agora que você tem uma cópia do projeto na sua conta, você precisa baixá-la para o seu computador. Isso é chamado de "clonar".

  1. Vá para a página do seu fork no GitHub (ex: github.com/SEU-USUARIO/nome-do-repositorio).
  2. Clique no botão verde "<> Code".
  3. Certifique-se de que a aba HTTPS está selecionada, pois o acesso SSH é bloqueado na faculdade.
  4. Copie a URL que aparece.

Agora, no seu terminal (dentro da pasta github), use o comando git clone:

git clone https://github.com/SEU-USUARIO/nome-do-repositorio.git

Isso vai criar uma nova pasta com o nome do repositório e baixar todos os arquivos para ela.

Passo 3 e 4: Fazendo e Salvando Alterações (add e commit)

Com o projeto no seu computador, você já pode começar a trabalhar.

  1. Navegue para a pasta do projeto no terminal: cd nome-do-repositorio
  2. Abra a pasta no seu editor de código (VS Code, Sublime, etc.) e faça as alterações necessárias: crie arquivos, edite, delete...

Depois de fazer suas modificações, você precisa "salvá-las" no histórico do Git. Este é um processo de dois passos:

1. Adicionar as mudanças (git add)

Este comando adiciona suas alterações a uma "área de preparação" (staging area). Pense nisso como colocar os itens que você quer salvar em uma caixa antes de fechar a caixa.

git add .

O ponto (.) significa "adicione todos os arquivos modificados nesta pasta e subpastas".

2. "Committar" as mudanças (git commit)

Este comando pega tudo que está na "área de preparação" e salva permanentemente no histórico do seu repositório local. É como fechar e etiquetar a caixa. Você deve sempre incluir uma mensagem clara descrevendo o que você fez.

git commit -m "Adiciona funcionalidade de login na página principal"

Sua mensagem deve ser curta e descritiva!

Passo 5: Enviando para o GitHub (push)

Até agora, suas alterações salvas (commits) estão apenas no seu computador. Para enviá-las para o seu repositório "forkado" no GitHub, use o comando git push.

git push origin master
  • origin é o nome padrão do seu repositório remoto (o seu fork no GitHub).
  • master é o nome da branch para a qual você está enviando.

Passo 6: Criando o "Pull Request" (PR)

O passo final! Um Pull Request é o seu pedido para que o dono do repositório original incorpore as suas mudanças.

  1. Vá para a página do seu fork no GitHub.
  2. Você provavelmente verá uma mensagem amarela com um botão "Contribute" ou "Pull request". Clique nele!
  3. Se não vir a mensagem, vá para a aba "Pull requests" e clique em "New pull request".
  1. Revise as suas alterações. O GitHub vai mostrar o repositório original à esquerda e o seu fork à direita.
  2. Escreva um título claro e uma descrição para o seu PR, explicando o que você fez e por quê.
  3. Clique em "Create pull request".

Pronto! Agora é só aguardar o feedback do dono do projeto.

Bônus: Mantendo seu Fork Sincronizado (pull)

Enquanto você trabalha, o repositório original pode ser atualizado. Para evitar problemas, é vital manter seu fork e sua cópia local atualizados. O jeito mais fácil é pelo próprio site do GitHub.

1. Sincronize seu Fork no GitHub

  1. Vá para a página principal do seu fork no GitHub.
  2. Clique no botão "Sync fork".
  3. No menu que aparecer, clique em "Update branch".

2. Atualize sua Cópia Local

Depois de sincronizar no site, as atualizações estão no seu repositório na nuvem, mas ainda não no seu computador. Use o comando git pull para baixá-las.

git pull

Faça isso sempre antes de começar a codificar para garantir que você está na versão mais recente!
