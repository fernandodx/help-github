Guia de Referência Shell (CLI)

Os Fundamentos do Terminal e da Linha de Comando.

Parte 1: O Que é o Shell?

Pense no seu sistema operacional (Windows, macOS) como uma casa:

  • Kernel (O Núcleo): A fundação, parte elétrica e hidráulica. Gerencia o hardware (CPU, memória). Você não interage diretamente com ele.
  • Shell (A "Casca"): O intérprete. É como você dá ordens ao Kernel. O Shell "traduz" seus comandos.

GUI vs. CLI

Shell Gráfico (GUI)

O que você usa 99% do tempo: janelas, ícones, mouse, menus (Ex: Windows Explorer, Finder do macOS).

Shell de Linha de Comando (CLI)

Interface baseada em texto onde você digita comandos. Você acessa o CLI usando um programa chamado Terminal.

Importante: O Terminal é apenas a janela; o Shell é o programa que roda dentro dela (como zsh, bash, ou powershell).

Por que o CLI é Tão Importante?

Se temos GUIs fáceis, por que aprender a "tela preta"?

  • Automação (Scripts): O motivo N° 1. Você pode escrever um "Shell Script" que executa 1.000 comandos em sequência. Você não pode "salvar" 1.000 cliques de mouse.
  • Eficiência: É muito mais rápido digitar mkdir minha_pasta do que dar 3 cliques para criar uma pasta.
  • Poder e Controle: Ferramentas avançadas (como Git) foram feitas para o CLI e oferecem mais opções que qualquer interface gráfica.
  • Acesso Remoto: Quase todos os servidores do mundo (Linux) não têm interface gráfica. A única forma de gerenciá-los é via Shell (usando SSH).

Parte 2: O Ambiente (macOS vs. Windows)

Seus alunos no Windows não podem usar os mesmos comandos que você (macOS) imediatamente.

  • Instrutor (macOS): Baseado em UNIX. Usa o app Terminal com o shell Zsh (que é compatível com Bash para comandos básicos).
  • Alunos (Windows): Não é baseado em UNIX.
    • Cmd.exe: O shell antigo. Comandos diferentes (ex: dir em vez de ls).
    • PowerShell: O shell moderno. Poderoso, mas com sintaxe diferente.

A Solução: Git Bash

Ao instalar o Git para Windows, ele inclui um programa chamado Git Bash.

O Git Bash emula o shell Bash (UNIX) dentro do Windows. Isso permite que seus alunos usem exatamente os mesmos comandos que você.

Instrução: Peça para seus alunos abrirem o "Git Bash" pelo Menu Iniciar.

Parte 3: Comandos Essenciais (Sintaxe UNIX)

Funcionará no macOS (Zsh/Bash) e no Windows (Git Bash).

Função Comando (UNIX / Git Bash) O que faz
Navegação pwd (Print Working Directory) Mostra em qual pasta você está.
Navegação ls (List) Lista arquivos e pastas no diretório atual.
Navegação cd <pasta> (Change Directory) Entra na pasta. Ex: cd Documentos
Navegação cd .. Sobe um nível de pasta.
Navegação cd ~ Vai direto para a sua pasta "Home" (pasta de usuário).
Manipulação mkdir <pasta> (Make Directory) Cria uma nova pasta. Ex: mkdir Projetos
Manipulação touch <arquivo> Cria um arquivo de texto vazio. Ex: touch notas.txt
Manipulação cp <origem> <destino> (Copy) Copia um arquivo. Ex: cp notas.txt backup/
Manipulação mv <origem> <destino> (Move) Move um arquivo. Também usado para renomear.
Manipulação rm <arquivo> (Remove) APAGA um arquivo. CUIDADO: Não tem Lixeira!
Manipulação rmdir <pasta> (Remove Directory) Apaga uma pasta vazia.
Manipulação rm -r <pasta> (Remove Recursivo) APAGA uma pasta e TUDO dentro dela. MUITO CUIDADO!
Visualização cat <arquivo> (Concatenate) Mostra o conteúdo de um arquivo de texto na tela.
Utilitário echo <texto> Imprime um texto na tela. Ex: echo "Olá"
Utilitário clear Limpa a tela do terminal.

Parte 4: Exercício Prático - "Estrutura do Curso"

Cenário: Criar uma pasta cursos_universidade. Dentro dela, semestre_1 e semestre_2. No semestre 1, criar java, flutter e web_fundamentos.

Digite estes comandos em ordem no seu Terminal / Git Bash:

# 1. Vá para sua pasta "Home" (lugar seguro) $ cd ~ # 2. Crie a pasta principal do projeto $ mkdir cursos_universidade # 3. Entre nessa pasta $ cd cursos_universidade # 4. Verifique se está no lugar certo $ pwd # 5. Crie as pastas dos semestres $ mkdir semestre_1 $ mkdir semestre_2 # 6. Liste o conteúdo (deve mostrar semestre_1 e semestre_2) $ ls # 7. Entre no primeiro semestre $ cd semestre_1 # 8. Crie as pastas das matérias (todas de uma vez!) $ mkdir java flutter web_fundamentos # 9. Crie um arquivo de plano de aula dentro de java $ touch java/plano_aula.txt # 10. Crie um arquivo de notas temporário $ touch notas_temp.txt # 11. Renomeie o arquivo de notas $ mv notas_temp.txt notas_finais.txt # 12. Mova esse arquivo para dentro da pasta flutter $ mv notas_finais.txt flutter/ # 13. Crie uma cópia desse arquivo em web_fundamentos $ cp flutter/notas_finais.txt web_fundamentos/ # 14. Ops, vamos apagar a cópia $ rm web_fundamentos/notas_finais.txt # 15. Limpe a tela $ clear # 16. Volte para a pasta cursos_universidade (subindo um nível) $ cd .. # 17. Apague a pasta semestre_2 (que está vazia) $ rmdir semestre_2 # 18. Liste o conteúdo (deve mostrar apenas semestre_1) $ ls

Como criar um arquivo README.md dentro de semestre_1/java/ com o texto "Aula de Java Swing" dentro dele, sem sair da pasta atual?
